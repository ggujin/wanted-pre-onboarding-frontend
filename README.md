# git commit convention
- `feat` - 신규 기능을 추가
- `refactor` - 코드 내부 구조를 변경
- `chore` - 실제 코드 동작과는 관련 없는 `config` or `script`
- `doc` - 문서에 관련된 작업
- `fix` - Bug 수정
- `style` - 코드 스타일 수정(Design 아닙니다!)
